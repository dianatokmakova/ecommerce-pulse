# Roadmap

Формат — доска задач: каждая неделя = спринт, задачи оформлены **тикетами** (`[ECP-XX]`)
с **Definition of Done (DoD)**. Ориентировочная нагрузка — 6–10 часов в неделю.
Каждый спринт опирается на предыдущий.

Технологический стек проекта: `SQL` · `ClickHouse` · `Python` · `Airflow` · `Docker` ·
`ECharts` · `DataLens` · `аналитика e-com` · `детекция аномалий`.

> В конце каждого спринта — коммит в GitHub и 2–3 предложения в README о проделанной работе.

---

## Спринт 0 — Фундамент и окружение (неделя 1)
**Цель:** поднять инфраструктуру и понять, как компоненты соединяются.

- **[ECP-01] Поднять стек в Docker.** `cp .env.example .env`, задать пароли, `docker compose up -d`.
  *DoD:* открываются ClickHouse `:8123/play`, MinIO `:9001`, Airflow `:8080`; `docker compose ps` — всё healthy.
- **[ECP-02] Разобраться в архитектуре.** Прочитать `docs/TECH_SPEC.md`, зафиксировать схему
  потоков данных. *DoD:* схему потоков данных можно описать за 1 минуту.
- **[ECP-03] Первый SQL в ClickHouse.** Создать тестовую таблицу, вставить строки, сделать `GROUP BY`.
  *DoD:* различие колоночной и строковой БД зафиксировано в README.

**Технологии:** Docker, ClickHouse basics, SQL. **Результат:** репозиторий запускается одной командой.

## Спринт 1 — Загрузка Olist и слой bronze (неделя 2)
**Цель:** реальные данные в складе.

- **[ECP-10] Загрузить Olist** с Kaggle, положить 9 CSV в MinIO bucket `raw/olist/`.
  *DoD:* файлы видны в консоли MinIO.
- **[ECP-11] Bronze-загрузка.** Через ClickHouse `s3()` загрузить CSV в `bronze.olist_*`
  (или запустить DAG `olist_batch_load` до шага bronze). *DoD:* `SELECT count()` по каждой таблице > 0.
- **[ECP-12] Профилирование данных.** Найти пропуски, странные значения, дубликаты (запросами).
  *DoD:* короткая заметка `docs/data_profiling.md` с находками.

**Технологии:** S3/MinIO, ClickHouse `s3()`, SQL-профилирование.
**Результат:** сырой слой по выгрузке маркетплейса с проверкой качества источника.

## Спринт 2 — Очистка: слой silver (неделя 3)
**Цель:** типизированные, чистые таблицы.

- **[ECP-20] Построить `silver.orders`.** Распарсить даты (`parseDateTimeBestEffort`),
  посчитать `delivery_days` и `delay_days`, флаг `is_delivered`. *DoD:* нет пустых `order_id`, даты — тип DateTime.
- **[ECP-21] Справочник категорий.** Склеить `products` с переводом PT→EN. *DoD:* у товаров английская категория.
- **[ECP-22] Остальные silver-таблицы** (customers, sellers, order_items, reviews). *DoD:* JOIN-ы между ними работают.

**Технологии:** SQL-трансформации, парсинг дат, JOIN, дедуп. **Результат:** типизированный слой silver.

## Спринт 3 — Витрины и e-com аналитика: слой gold (неделя 4)
**Цель:** ответить на бизнес-вопросы цифрами.

- **[ECP-30] Выручка/AOV** (`gold.daily_revenue`). *DoD:* видно топ категорий и динамику по месяцам.
- **[ECP-31] RFM-сегментация** (`gold.rfm`). *DoD:* есть сегменты Champions/At Risk/… и список «кого реактивировать».
- **[ECP-32] Когортный retention** (`gold.cohort_retention`). *DoD:* матрица когорт строится; видно, где проседает.
- **[ECP-33] SLA доставки** (`gold.delivery_sla`). *DoD:* доля просрочек по штатам; гипотеза о связи с отзывами.

**Технологии:** оконные функции, квантильные скоры, когортный анализ, бизнес-интерпретация.
**Результат:** витрины для продуктовой и операционной аналитики, 3 инсайта.

## Спринт 4 — Оркестрация в Airflow (неделя 5)
**Цель:** пайплайн запускается автоматически, а не вручную.

- **[ECP-40] Включить DAG `olist_batch_load`.** Прогнать end-to-end bronze→silver→gold. *DoD:* зелёный ран в Airflow.
- **[ECP-41] Data-quality проверки.** Убедиться, что таск `data_quality_checks` падает при «битых» данных
  (внести намеренную ошибку — таск становится красным). *DoD:* видно, как Airflow сигнализирует о проблеме.
- **[ECP-42] Расписание и ретраи.** Настроить `schedule`, `retries`, backfill. *DoD:* расписание и ретраи отрабатывают.

**Технологии:** Airflow (DAG, расписание, зависимости, DQ). **Результат:** пайплайн обновляется автоматически.

## Спринт 5 — Живой поток цен + аномалии (недели 6–7)
**Цель:** стриминг и ML-детекция аномалий.

- **[ECP-50] Задеплоить Fake API на Vercel.** Папка `fake-marketplace-api/`. *DoD:* `/api/prices` отдаёт JSON, ссылка в `.env`.
- **[ECP-51] Hourly-ingest.** Включить DAG `live_pricing_ingest`: API+FX → Parquet в MinIO → `bronze.pricing_raw` → `silver.pricing`.
  *DoD:* за сутки накопилось ≥24 снимка; данные растут сами.
- **[ECP-52] Детекция аномалий.** `detect_anomalies.py`: z-score + IQR + IsolationForest → `gold.pricing_anomalies`.
  *DoD:* подмешанные скачки цен ловятся; в README сравнение «какой метод даёт меньше ложных срабатываний».
- **[ECP-53] Обогащение.** Подмешать курс валют (Frankfurter) и/или погоду (Open-Meteo) как фичу. *DoD:* в данных есть `price_usd`.

**Технологии:** httpx/API, FastAPI+Vercel, потоковая загрузка, pandas, scikit-learn, статистика.
**Результат:** мониторинг динамического ценообразования с авто-детекцией аномалий.

## Спринт 6 — BI-слой: DataLens + свой дашборд (неделя 8)
**Цель:** результат, доступный по ссылке.

- **[ECP-60] DataLens.** Подключить к ClickHouse (read-only `bi_reader`, whitelist IP), собрать дашборд
  (выручка, RFM, когорты-heatmap, карта, аномалии), опубликовать. *DoD:* публичная ссылка работает.
- **[ECP-61] Свой дашборд.** Поднять `dashboard-api` (FastAPI, read-only поверх gold), фронт на ECharts (`dashboard/`)
  задеплоить на Vercel. *DoD:* дашборд открывается по ссылке и тянет данные из gold через `dashboard-api`.
- **[ECP-62] Безопасность.** HTTPS через Caddy, закрыть консоли, проверить, что наружу торчит только нужное.
  *DoD:* `8123` не висит голым в интернете.

**Технологии:** DataLens, ECharts, FastAPI, сетевая безопасность. **Результат:** 2 живых дашборда.
